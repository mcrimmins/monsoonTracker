# Southwest Monsoon Tracker

## Project Overview
This project tracks, analyzes, and visualizes the North American Monsoon (NAM) over the desert Southwest and northern Mexico. It processes 35 years of ERA5 reanalysis into daily and seasonal diagnostics — the 500 hPa monsoon ridge (geopotential height), low-level moisture surges (specific humidity), and total column water vapor (TCWV) — and publishes them as the **NAM Diagnostic Hub**, a Quarto website in `NAM_tracker/`.

## Dataset: Copernicus ERA5 Reanalysis
Data is sourced from the Copernicus Climate Data Store (CDS): [ERA5 hourly data on pressure levels](https://cds.climate.copernicus.eu/cdsapp#!/dataset/reanalysis-era5-pressure-levels) and the single-levels product for TCWV.

* **Temporal Resolution:** Hourly data collapsed to 4x-daily slices (00/06/12/18Z), averaged to daily values.
* **Timeframe:** 1991 - 2025 (monsoon season: June through October, 153 days/year).
* **Spatial Extent:** Longitude -125 to -95, Latitude 15 to 45 (US Southwest and northern Mexico).
* **Target Variables:** Geopotential (`z`), Specific Humidity (`q`), Total Column Water Vapor (`tcwv`).
* **Pressure Levels:** 250, 500, 700, and 850 hPa.
* **Climatological baseline:** 1991-2025, 5-day rolling smoothed.

## Scripts & Pipeline
All scripts assume the working directory is the project root (`monsoonTracker/`).

### Data acquisition & processing
* **`config.R`** - CDS API setup, target years/months, bounding box, variables.
* **`src/01A_batch_download_era5.R`** / **`01B_sequential_download_era5.R`** - Download raw hourly NetCDFs via `{ecmwfr}`, handling CDS rate limits and skipping already-downloaded months.
* **`src/02_daily_stat_processing.R`** - Collapses hourly data into 4x-daily slices, then a daily mean per variable/level, with unit conversion (geopotential -> meters, specific humidity -> g/kg).
* **`src/03A_build_pressure_climatology.R`** / **`03B_build_tcwv_climatology.R`** - Build the 1991-2025 calendar-day climatology (raw + 5-day smoothed) for all 8 pressure-level variables and TCWV.
* **`src/04_calculate_daily_anomalies.R`** - Subtracts the smoothed climatology from each year's actual daily values to produce per-year anomaly stacks (153 layers x 9 variables x 35 years).

### Figure generation
* **`src/05B_animate_dashboard_v2.R`** - `generate_synoptic_frame()`: the daily two-panel dashboard (observed state + anomaly), with two-pass focal-smoothing "H" ridge tracking. Batch-generates one PNG per day per year into `NAM_tracker/output/frames/<year>/`.
* **`src/06_plot_climo.R`** / **`06B_plot_climo_monthly.R`** - The 1991-2025 climatology as a 5x5 pentad grid and a 1x5 monthly grid (`NAM_tracker/output/climatology_*.png`).
* **`src/08_hovmoller.R`** - `generate_hovmoller()`: date-by-latitude Hovmoller diagram of zonal-mean TCWV across the Gulf of California corridor (115-105degW), observed vs. climatological 25 kg/m2 threshold, with a locator inset map. Output in `NAM_tracker/output/hovmoller/`.
* **`src/09_seasonal_summary.R`** - `generate_seasonal_summary()`: full-season mean-state + anomaly-footprint maps with a daily ridge trajectory overlay. **Parked** - the 153-point trajectory renders too noisy to be useful; revisit with precipitation data to characterize seasonal anomalies instead. Output in `NAM_tracker/output/seasonal_summary/`.
* **`src/10_pentad_year.R`** - `generate_pentad_year()`: per-year companion to script 06 - the same 5x5 grid layout, but for one year's actual absolute values and true pixel-level anomalies vs. baseline. Output in `NAM_tracker/output/pentad/`.

### Website generation
* **`src/07_create_year_pages.R`** - Generates `NAM_tracker/years/<year>.qmd` for 1991-2025. Each page is a tabbed panel (Daily Synoptic Player / Season Hovmoller / 5-Day Evolution) that checks for its underlying image(s) at render time and shows a "data pending" callout instead of a broken image if they don't exist yet. Re-run this after regenerating figures for more years - existing pages also just pick up new images automatically on next render/preview without rerunning it.

## The website (`NAM_tracker/`)
A Quarto website (`quarto preview` or `quarto render` from within `NAM_tracker/`) with:

* **Home** - overview + quick links.
* **Monsoon Basics & Climatology** - what TCWV/Z500/Q700/"H" mean, the 1991-2025 baseline, and the climatology grids.
* **Historical Seasons** - a card-grid index of all 35 years, plus the 5 most recent years as direct navbar links; each year page has the Daily Player, Hovmoller, and 5-day evolution tabs.
* **Data & Methods** - full write-up of the data source, processing pipeline, ridge-detection algorithm, and diagnostic definitions.

Site theme (`styles.css`) pulls its accent colors from the same TCWV/anomaly palettes used in the figures.

## Repository notes
`data/` and `NAM_tracker/output/` are gitignored - both are fully regenerable from the pipeline scripts above and are too large to track (raw + processed data alone is ~34 GB; daily frames for all 35 years would add several more GB). Clone the repo, run the pipeline, then `source("src/07_create_year_pages.R")` and `quarto preview` from `NAM_tracker/` to reproduce the full site locally. `.Renviron` (CDS API key) is also gitignored and must be created locally - see `config.R`.
